-port: specifies the port you wish to run the Server on (default is 4444)

-timeout: specifies the number of seconds that the server will wait for the browser to respond to the previous command, or for the client driver to issue a new request.

-interactive: starts the server in Interactive mode. In this mode you can type Selenese commands on the command line (e.g. cmd=getNewBrowserSession&1=*firefox&2=http%3A%2F%2Fwww.google.com).

-htmlSuite: automatically runs a single HTML suite and saves the results to a file. See the Selenese section for more information.

-Dhttp.proxyHost and -Dhttp.proxyPort: Selenium RC normally overrides your proxy server configuration, using the Selenium Server as a proxy. Use these options if you need to use your own proxy together with the Selenium Server proxy. (Note that these settings are new for Selenium 0.8.0, and may be a bit raw... use them with care, and report any issues to our user forums.) Use the proxy settings like this:

java -Dhttp.proxyHost=myproxy.com -Dhttp.proxyPort=1234 -jar selenium-server.jar

-Dselenium.javascript.dir: override Selenium Core's javascript location. Without this system property, Selenium RC will use Selenium Core pre-embbed in the jar. The following command line will start Selenium Server using Selenium Core at c:\selenium-core\

java -Dselenium.javascript.dir=c:\selenium-core\code\javascript -jar selenium-server.jar
